acme.sh 自动颁发更新免费HTTPS证书

 2022.12.7  2022.12.7  工具/部署  1082  3 分钟

Acme.sh 是一款优秀免费的HTTPS证书管理器,2020年部署过一次，直到今天从未出现过任何异常，稳定高效。

最近新添加了域名，所以又操作了一次，发现忘的差不多了，因此温故而知新。中文官方文档

国内的服务器在无法翻的情况下怎么安装？

官方给的方法是通过gitee下载安装，但是当我尝试，需要登录等问题，觉得比较麻烦。通过分析结构，我采用了Copy的另类方式安装

  • 在本地WSL系统上，export http_proxy... 本地代理 执行安装脚本，注意安装用户
  • 此时 ~/.acme.sh 文件下包含所要的全部文件 tar 打包.acme.sh
  • 上传到目标服务器同目录结构下，如果没有权限，则 chmod 777 ~/.acme.sh/acme.sh 权限
  • alias acme.sh=~/.acme.sh/acme.sh 不出意外就安装成功了。 acme.sh
  • 注意此方法安装证书，并不会默认更新证书， 所以要执行 acme.sh --install-cronjob

├── http.header
# 一些操作可以选择进行通知设置，比如支持 mail dingtalk smtp 等等
└── notify

设置默认CA 为 letsencrypt

zeroSSL 需要邮箱，还未去了解，所以就直接采用以前的 letsencrypt

acme.sh --set-default-ca --server letsencrypt

如何添加 泛域名证书 例如 *.bbdshow.top

通常我在比较懒的情况下，希望配置一次就满足以后的所有扩展需求。所以HTTPS证书配置常有泛域名需求。毕竟泛域名的证书收费实在是太贵了！

此处目前只记录 自动DNS方式 域名服务商 阿里云 其他域名服务商，也介绍了相关方法

# Ali_Key 与 Ali_Secret 在阿里云控制后台获取, 不同DNS服务商环境变量Key是不同的，如何确认？ 
# 找到 ~/.acme.sh/dnsapi/dns_xx.sh 前几行有明确读取环境变量的代码
export Ali_Key="xxx"
export Ali_Secret="xxx"
# 这里注意 dns_ali 就代表调取 dnsapi/dns_ali.sh 其他DNS服务商可以 dnsapi 这个目录里面查询具体的命名，例: cloudflare = dns.cf.sh
# 这里注意 泛域名 * 这里一定要加 '' 保证正常解析
acme.sh --issue -d '*.bbdshow.top'  --dns dns_ali

# 当你还需要第二次添加域名,如果是同一DNS服务商，就不需要 export 了，相关的 DNS Key 会保存在 account.conf 文件内

acme.sh --install-cert -d '*.bbdshow.top' \
--key-file       /etc/nginx/ssl/bbdshow.top.key \
--fullchain-file /etc/nginx/ssl/bbdshow.top.crt \
--reloadcmd     "service nginx force-reload"

# 会把相关证书写入到 /etc/nginx/ssl 后期 crontab 更新安装证书同理
# service nginx force-reload 如果是 systemctl 命令会自动替换成 /bin/systemctl force-reload nginx.service

# 执行之后 此时 ~/.acme.sh/*.bbdshow.top/ 文件内，就是域名的所有信息
acme.sh --info -d '*.bbdshow.top'
